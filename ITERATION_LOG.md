# Iteration Log - Habit Tracker MVP

## Iteration 1
**Problem**: The initial prototype did not update the progress counter when habits were marked complete.

**Prompt**: 
"Fix the JavaScript so the progress counter updates correctly when a habit is marked as complete."

**Result**: 
The progress counter now updates whenever a habit is completed or uncompleted.

**Status**: ✅ Fixed


## Iteration 2
**Problem**: Users could add empty habits by clicking the Add button without typing anything.

**Prompt**: 
"Add validation so users cannot add an empty habit and show an error message."

**Result**: 
The app now prevents empty input and shows an alert if the field is blank.

**Status**: ✅ Fixed


## Iteration 3
**Problem**: The user interface looked too plain and not professional.

**Prompt**: 
"Improve the CSS styling to make the app look modern with better layout and colors."

**Result**: 
Added a gradient background, better spacing, and styled buttons for a cleaner design.

**Status**: ✅ Fixed


## Iteration 4
**Problem**: It was difficult to distinguish completed habits visually.

**Prompt**: 
"Make completed habits visually different from incomplete ones."

**Result**: 
Completed habits now show a strikethrough and gray color.

**Status**: ✅ Fixed


## Iteration 5
**Problem**: There was no way to remove habits once added.

**Prompt**: 
"Add a delete button for each habit so users can remove them."

**Result**: 
Each habit now includes a delete button that removes it from the list.

**Status**: ✅ Fixed


## Iteration 6
**Problem**: Buttons looked cluttered with text labels.

**Prompt**: 
"Replace text buttons with simple icons to improve UI clarity."

**Result**: 
Buttons now use ✔ and ✖ icons for a cleaner look.

**Status**: ✅ Fixed


## Reflection
Throughout the iterations, I learned how to:
- Identify bugs by testing features step-by-step
- Use AI to fix specific problems instead of rewriting everything
- Improve both functionality and design incrementally

The biggest challenge was debugging JavaScript behavior, but breaking problems into small steps made it manageable.
